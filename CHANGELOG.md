# 更新日志

## [未发布]

### 新增

- 可在界面”分析规范”中选择独立的 VDA 19.1 模式（阶段 1），原有自定义模式保持不变
- 增加 E–N 标准尺寸等级和 `[下限, 上限)` 分档边界
- 增加 50 μm 至少覆盖 10 像素的强制分辨率检查
- **颗粒宽度（Feret min）测量**：新增 `_minimum_feret_diameter` 旋转卡壳算法，同时计算每颗颗粒的宽度（最小 Feret 径）与长度（最大 Feret 径），符合 VDA 19.1 §8.2.2 要求
- 颗粒记录与 CSV 输出新增 `width_px`/`width_um` 字段
- 单图、批处理、CSV、JSON 和 PDF 结果记录分析模式与分辨率证据
- 黄色比例尺改为测量两条竖线最外侧边缘的距离，并在分析元数据中记录外沿坐标

### 修复

- **VDA 模式边界处理**：修复 `should_report` 中 `configured_minimum_um=0.0` 因 Python `or` 运算符的 falsy 行为被错误替换为模式默认最小值的问题

### 约束

- VDA 模式禁止通过手工输入尺寸新增或拆分颗粒，仅允许删除误检和撤销
- 明确标注当前为部分标准化实现，不构成完整 VDA 19.1 合规声明

### 版本

- 软件版本更新为 `1.1.0`
- 分析算法版本更新为 `2.2.0`

## [1.0.1] - 2026-06-30

### 修复

- **英文 Windows 兼容性**：修复英文 Windows 系统下导入图片后点击"开始统计"无响应的问题
  - 使用 `np.frombuffer` + `cv2.imdecode` 替代 `cv2.imread`，使用 `cv2.imencode` + `write_bytes` 替代 `cv2.imwrite`，避免 Windows C 运行时 `fopen` 对 UTF-8/Unicode 路径不兼容
  - HTTP 服务增加异常捕获，防止客户端断连时异常逃逸导致线程静默死亡
  - 前端增加请求超时和空值防护，兼容 TIFF/BMP 等图片格式
- **扫描动画**：修复图片放大后扫描动画被截断的问题。暗色遮罩现在覆盖整张放大后的图片，扫描线保持 100% 视图时的视觉比例并填满显示窗口

### 测试

- 新增 `test_unicode_path_read_and_write` 回归测试，覆盖中文/Unicode 路径下的图片读取与结果写入
- 测试总数从 27 个增加到 **28 个**，全部通过

### 构建与配置

- 跟踪 `颗粒度计数台.spec`，确保 PyInstaller 打包配置一致
- 修正 `.gitignore`：移除对 `*.spec` 的错误忽略，新增对自动生成的 `skills-lock.json` 的忽略
- `SOFTWARE_VERSION` 更新为 `1.0.1`

## [1.0.0] - 2026-06-27

### 初始版本

- 本地处理清洁度滤膜照片，自动识别右下角黄色比例尺
- 按颗粒最大 Feret 径分四档统计：25–50、50–100、100–200、＞200 μm
- 支持 JPG、PNG、TIFF、BMP 格式
- 可调整椭圆统计区域，圆边自动内缩保护带
- 生成彩色标注图、汇总 CSV、逐颗粒明细、PDF 报告和 ZIP 结果包
- 人工复核：新增、删除、拆分颗粒，支持撤销
- 参数模板保存与快速切换
- 批量处理文件夹
- 支持构建 Windows `.exe` 和 macOS `.app`
